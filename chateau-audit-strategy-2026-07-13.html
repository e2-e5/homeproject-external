Вопросы для аудита бизнеса

Достаточно ответов «в процентах» / «диапазоне» / «примерно» — по каждому нам надо понять масштаб и точки роста. Полные цифры не нужны.

💰 Блок 1. Финансы и обороты (6 вопросов)

оборот

1.1. Каков годовой оборот компании? (можно диапазоном: 100–200 млн / 200–500 млн / 500 млн+)

Зачем: понять размер бизнеса, сравнить с нашими бенчмарками и оценить потенциал роста от AI-внедрения (обычно 15–30% при полной реализации).
канал

1.2. Как оборот распределён между каналами? (шоурумы % / сайт % / B2B-партнёры %)

Зачем: определить где AI даст максимальный эффект. Если сайт даёт 5% — там огромный потенциал; если 50% — фокус на удержании и upsell.
маржа

1.3. Средняя маржинальность по трём категориям (бытовая техника / посуда / мебель) — в процентных диапазонах

Зачем: понять экономическую отдачу от AI-инвестиций в каждую категорию, определить приоритет.
чек

1.4. Средний чек по каналам: онлайн-заказ / офлайн / B2B-проект (порядок цифр)

Зачем: рассчитать сколько денег принесёт каждый лид/конверсия — обосновать ROI внедрения.
рост

1.5. Динамика оборота 2024 → 2025 → 2026 (в % — рост/падение)

Зачем: понять фазу бизнеса. Рост = инвестируем в масштабирование, стагнация = ищем новые источники, спад = фокус на эффективность.
конверсия

1.6. Средняя конверсия сайта в заказ (в %, если считаете)

Зачем: базовый KPI для замера эффекта AI-консультанта — типично улучшение x1.5–x2.5 от базовой.

B2C/B2B

Зачем: сегментация коммуникации, разные шаблоны КП и цикла сделки для каждого сегмента.

Зачем: определить масштаб рассылок, необходимость сегментации, потенциал повторных продаж.

Зачем: если LTV высокий — фокус на retention-механики (нужны напоминания, персональные предложения); если низкий — фокус на acquisition.

2.4. Основные источники лидов (сайт %, соцсети %, «сарафан» %, реклама %, дизайнеры %)

Зачем: понять куда направить усилия AI (email/соцсети/consult) в первую очередь.

Зачем: понять текущие процессы. Если ручные — экономия от автоматизации огромна. Если автомат — интеграция уже есть.
поставщики

Зачем: критично для актуальности сайта. Наш опыт: 90% нишевых игроков не имеют — здесь легко дать быстрый эффект.

Зачем: наш content-creator агент способен закрыть 80% работы SMM-специалиста. Оценить экономию.

4.4. Есть ли регулярные email-рассылки? На какой сервис? Open rate / CTR?

Зачем: если рассылок нет — огромный неиспользуемый актив. Если есть — знаем базовый бенчмарк для сравнения после внедрения AI.

2

Платформа: текущая vs Medusa

Техническое сравнение с точки зрения возможности внедрения AI-агентов.

Что видно по chateaudevaisselle.com сейчас

На сайте — классическая CMS-система (тип Bitrix / 1С-Битрикс / OpenCart, точно определить можно после интервью с IT). Признаки: серверный рендеринг всех страниц, тяжёлая админка, отсутствие headless-режима, ограниченный REST API. Это стандартный «монолит» — работает, но плохо адаптируется под современные надстройки.

ПараметрТекущая платформаMedusa (наш стек)
Архитектура Монолит: фронт + бэк слиты. Любое изменение фронта требует правки в CMS. Headless: витрина отделена от логики. Можно менять дизайн без затрагивания каталога, ставить несколько storefront'ов.
API для AI Ограниченное REST-API либо вообще XML-выгрузка. AI-агент читает медленно, писать почти не может. Современное REST + GraphQL из коробки. AI-агент читает/пишет любое поле, создаёт товары, меняет цены, управляет коллекциями.
Скорость обновлений каталога 1С-обмен раз в 3–24 часа. Ручные правки через тяжёлую админку. Real-time через API. Обновление цены/остатка → мгновенно на сайте. AI-агент дёргает автоматически.
Кастомизация фронта Тема шаблонная, любые правки → часы работы верстальщика. Next.js/React-фронт, современный UX. Правки — минуты в компоненте.
Интеграции Только через готовые модули. Кастомная интеграция — заказ разработчику CMS. Любая через API: CRM, Unisender, Telegram, платежи, доставка, склад.
Стоимость владения Лицензии CMS + доработки поштучно. Обычно 200–500К ₽/год минимум. Open-source ядро. Основная стоимость — хостинг (10–30К ₽/мес) + наша поддержка.
Поиск и AI-подсказки Базовый MySQL LIKE / встроенный поиск CMS. Векторный поиск невозможен. Из коробки: pgvector, семантический поиск. AI-консультант находит «нужна плита с духовкой и грилем до 100К» без exact-match.
Масштабирование При росте трафика — вертикально (более мощный сервер). Дорого. Горизонтально. Каждый компонент масштабируется отдельно.
Скорость загрузки страниц Типично 3–7 секунд (полный SSR каждой страницы). 0.5–1.5 сек (SSG + edge caching).

💡 Главный вывод по платформе

Текущая CMS справляется с базовой e-commerce задачей, но физически ограничивает возможности AI-агентов. Мы можем «подключить» AI сверху (чат-виджет, парсинг каталога), но полная автоматизация — создание карточек, автообновление цен, real-time рекомендации — требует другой архитектуры.

3

Три сценария внедрения AI

Клиент задал ключевой вопрос: «внедрять на текущий сайт или мигрировать на Medusa?». Ответ: есть три пути, у каждого своё место.

Сценарий А. AI-агенты поверх текущего сайтаБыстро, но ограниченно

Не трогаем текущий сайт. Ставим сверху: чат-консультант (виджет), email-агент, SMM-агент. Каталог читаем через парсинг или существующий XML-экспорт (обновление раз в сутки).

Плюсы
  • Запуск за 2–3 недели
  • Не рискуем текущим сайтом (продажи не падают)
  • Быстрый эффект по SMM и email
  • Низкие вложения (40–150К ₽ первый месяц)
Минусы
  • Консультант работает на «вчерашних» остатках
  • Автосоздание карточек невозможно — только рекомендации менеджеру
  • Цены обновляются с задержкой (после следующего 1С-обмена)
  • Riski: любое изменение CMS ломает парсер

⚠️ Ключевой риск сценария А

AI-консультант обещает клиенту товар, которого уже нет (устаревшие остатки). Один такой инцидент = удар по репутации. Решается только real-time API — который в текущей платформе появится только после доработки за 200–500К ₽ (силами разработчика их CMS).

Сценарий Б. Полная миграция на Medusa за один разДолго и рискованно

Одномоментно переносим весь сайт (все категории, весь каталог, все интеграции) на Medusa. Старый сайт отключаем.

Плюсы
  • Единая современная платформа
  • Полная функциональность AI сразу
  • Максимальный эффект по всем метрикам
Минусы
  • 6–9 месяцев работы
  • Риск потерять SEO-позиции при массовой смене URL
  • Крупные единовременные вложения (1.5–3 млн ₽)
  • Ошибка в переносе одной категории → падает весь сайт
  • Долгое ожидание первого эффекта

✅ Сценарий В. Поэтапная миграция по категориямРекомендуем

Начинаем с одной категории (бытовая техника) на Medusa, параллельно текущий сайт продолжает работу. Через 6–8 недель — первая категория на Medusa с полным набором AI-агентов. Затем поочерёдно переносим посуду и мебель. Никакой единовременной ломки, каждый этап окупается.

Плюсы
  • Первый эффект уже через 6–8 недель
  • Каждый этап окупается до следующего
  • Постепенная миграция базы клиентов и партнёров
  • Команда клиента учится работать с новой системой не в стрессе
  • SEO переносится плавно (301-редиректы категорий)
  • Можно остановиться на любом этапе, если что-то не подходит
  • Умеренная стоимость на старте (150–400К ₽ на MVP)
Минусы
  • Временно 2 системы работают параллельно (админам нужно обучение)
  • Единый чекаут — либо на старом, либо на новом (нужно решать в первом этапе)
  • Общий срок дольше сценария Б

Почему именно бытовая техника первой

1) Самая высокая маржа в категории. 2) Самая сложная для клиента категория (техник помогает больше всего). 3) Совпадает с нашей экспертизой — HOMEPROJECT сам работает в этой нише, мы уже знаем поставщиков, каталог, тонкости КП.

4

Этапы работ по сайту — 6 фаз

Реалистичный план для рекомендованного сценария В. Общий срок «сайт полностью на Medusa + все AI-агенты работают» — 4–6 месяцев.
1

Discovery и архитектура1–2 недели

Фундамент. Без него любой следующий этап рискует пересобираться.
  • Интервью с командой: продажи, маркетинг, IT, склад
  • Аудит каталога: выгрузка + инвентаризация SKU, брендов, категорий
  • Согласование архитектуры (URL, домены, subdomain vs path)
  • Дизайн-система (частично на базе текущего сайта, частично наши компоненты)
  • План интеграций: 1С / складская программа / платежи / доставка
  • Выбор пилотной категории и её объём
2

MVP: бытовая техника на Medusa5–6 недель

Первая работающая витрина. Отдельный субдомен, например appliances.chateaudevaisselle.com, ссылки с основного сайта переводят на неё.
  • Развёртывание Medusa на инфраструктуре клиента (или нашей)
  • Импорт каталога БТ (~2500 SKU) из текущей CMS
  • Интеграция с 1С: обмен остатками и ценами (real-time или интервально — по выбору)
  • Разработка/адаптация фронта Storefront в стилистике chateaudevaisselle
  • Настройка платежей и доставки
  • Миграция SEO по категории (301-редиректы, sitemap)
  • Обучение контент-менеджера работе с админкой Medusa
3

AI-агенты для витрины БТ2–3 недели, параллельно с фазой 2

Как только Medusa работает — накладываем интеллект.
  • AI-консультант на сайте: чат в правом нижнем углу, понимает запросы вроде «нужна плита с духовкой и грилем до 120К» → возвращает 3–5 моделей с обоснованием
  • Механика КП (Proposal Writer): клиент/дизайнер собирает корзину → получает PDF-КП с брендированием, ценами, характеристиками, срокам доставки
  • Автоматизация карточек: агент вычитывает прайс-листы поставщиков, автоматически создаёт/обновляет карточки, флагирует спорные моменты для менеджера
4

CRM: сегментация базы и коммуникация3–4 недели

Работа с базой клиентов и партнёров. Начинать можно параллельно с фазой 2.
  • Импорт базы клиентов и партнёров в единую CRM (у нас — PostgreSQL + типизация)
  • Сегментация: B2C по чеку и категориям, B2B по типу (дизайнеры / застройщики / дилеры), по активности
  • Настройка email-рассылок через Unisender: welcome-серия, реактивация, релиз новинок
  • Telegram-канал для B2B партнёров с уведомлениями (новые остатки, акции)
  • Партнёрская программа: единый кабинет для дизайнеров с прайс-листом и корзиной
5

Соцсети под управлением AI2 недели

Content-creator агент. Начинать можно после фазы 3.
  • Content-plan на месяц: новинки, акции, обзоры, «полезное»
  • Автогенерация постов TG / VK / Instagram под каждую платформу
  • Визуалы: карточки-обложки с ценой и главным преимуществом
  • Согласование через chat-интерфейс: SMM-менеджер получает 5 черновиков утром, апрувит одной кнопкой
  • Аналитика: авто-отчёт по метрикам раз в неделю
6

Расширение: посуда → мебель → полная замена сайта4–8 недель на категорию

Повторяем цикл фаз 2–3 для оставшихся категорий.
  • Приоритет 2: посуда (2500–4000 SKU, зависит от объёма). Сроки — короче, потому что каркас уже готов
  • Приоритет 3: мебель (сложнее в карточках — размеры, комплектация, фото)
  • Финал: единый storefront под chateaudevaisselle.com, старая CMS уходит в архив
  • Единый checkout, единый профиль клиента, единая CRM

💰 Экономика по этапам

Setup (Discovery + MVP + AI для БТ): ~150–250К ₽. Далее — фиксированный тариф 40–55К ₽/мес за поддержку и развитие. Каждое расширение категории — от 80К до 150К ₽ setup + без изменения абонплаты. Пилот показывает эффект за 2–3 месяца, дальше идёт органический рост.
