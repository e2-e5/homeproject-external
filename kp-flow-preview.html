Шаг 4 — proposal-writer агент генерирует и отправляет КП
1

Сборка KpInput из kp_requests

contactId + brands[0] → brand; categories + description → brief; budget, finish → briefStructured

~1 сек
2

Поиск SKU в Medusa по бренду + категориям

GET /store/products?brand=Küppersbusch&q=духовой → топ-3 SKU → variants[].items[]
Если SKU не найден точно → fuzzy-поиск по названию (нужен инженеру P0)

~3–5 сек
3

Генерация текста (LLM)

Быстрое: 2 поля — introTitle + introText (writeProse format=fast)
Полное: ~25 полей — brief, варианты A/B, визуальные карточки, рекомендация

Быстрое: ~15 сек Полное: ~80 сек
4

Рендеринг HTML → отправка клиенту

kp-render-simple.ts или kp-render-full.ts → HTML → email + уведомление менеджеру

~2 сек
5

Обновление статуса в kp_requests

status = 'sent', kpId, kpUrl, sentAt = now()

~1 сек
